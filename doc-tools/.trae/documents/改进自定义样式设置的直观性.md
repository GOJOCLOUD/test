# 改进自定义样式设置的直观性

## 问题分析
当前系统中，用户在选择自定义格式时，需要手动输入字体名称和字号，不够直观。用户希望能像Word一样，直接从下拉菜单中选择常用字体和字号。

## 改进方案
1. **将字体输入框改为下拉选择框**
   - 在前端组件中添加常用字体列表
   - 包含中文字体（如宋体、黑体、微软雅黑等）和英文字体（如Times New Roman、Arial等）
   - 保持输入框的可编辑性，允许用户输入自定义字体

2. **将字号输入框改为下拉选择框**
   - 添加Word常用字号列表（如小四、五号、四号、三号等）
   - 对应Word中的字号与磅值映射关系
   - 保持输入框的可编辑性，允许用户输入自定义磅值

3. **优化样式设置界面**
   - 统一字体和字号的选择样式
   - 确保界面美观易用

## 实现步骤
1. 修改 `frontend/src/components/BeautifyPanel.jsx` 文件
   - 在自定义样式设置部分，将字体和字号的 `<input>` 标签改为 `<select>` 标签
   - 添加常用字体列表
   - 添加常用字号列表
   - 保持组件的原有功能不变

2. 测试改进后的功能
   - 确保下拉选择框正常工作
   - 确保手动输入功能仍然可用
   - 确保样式设置能正确应用到文档美化中

## 预期效果
- 用户可以直接从下拉菜单中选择常用字体和字号，无需手动输入
- 界面更加直观易用，符合用户使用Word的习惯
- 保持了原有的灵活性，允许用户自定义字体和字号

## 技术要点
- 使用React的state管理下拉选择的值
- 保持原有表单提交逻辑不变
- 确保中文字体和英文字体都能正确显示和应用
- 正确映射Word字号与磅值的对应关系
